# Frontend Design

Approach this as the design lead at a small studio known for giving every client a visual identity that could not be mistaken for anyone else's. Make deliberate, opinionated choices about palette, typography, and layout specific to this brief, and take one real aesthetic risk you can justify.

## Product ground (Finança Clara)

- **Subject**: gestão financeira assistiva — organização e orientação, não banco.
- **Audience**: pessoas que querem clareza e calma, não jargão financeiro.
- **Job**: ajudar a saber o próximo passo; o gestor orienta, o cliente age.
- **Tone**: acolhedor, confiável, claro. Sem linguagem punitiva. Sem look bancário técnico.

## Design principles

- The hero is a thesis — open with the most characteristic thing (brand, next action, calm clarity).
- Typography carries personality: pair display + body deliberately; avoid default stacks (Inter, Roboto, Arial, Geist-as-everywhere).
- Structure is information, not decoration. Don't use 01/02/03 unless order is meaningful.
- Motion: 2–3 intentional moments. Respect `prefers-reduced-motion`.
- Spend boldness in one signature; keep surroundings quiet.

## Avoid (AI defaults)

Do not default to: (1) warm cream #F4F1EA + terracotta + high-contrast serif; (2) near-black + acid green/vermilion; (3) broadsheet hairline rules / zero radius / dense columns; (4) purple-on-white / purple-indigo gradients.

## Process

1. Name subject, audience, single job.
2. Draft token system: 4–6 named hex colors, type roles, layout concept, one signature.
3. Critique: if it could belong to any SaaS after removing the brand, revise.
4. Build from tokens only. Critique again (screenshot when possible). Remove one accessory.

## Finança Clara — current direction "Clareira"

- **Palette**: Mist `#E8F0EC`, Cloud `#F7FAF8`, Ink `#1A2E28`, Canopy `#1F6B56`, Leaf `#3D9B7A`, Soft clay `#C4A574` (accent only).
- **Type**: Syne (display/brand) + Manrope (body/UI).
- **Signature**: soft drifting canopy-light band behind the brand (light through a clearing).
- **Thesis screens**: login = brand first; dashboard = "O que fazer agora" before metric grids.
